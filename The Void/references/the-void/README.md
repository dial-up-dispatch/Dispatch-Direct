# the void

*received. the void remains unmoved. so can you.*

The Void is a single-purpose release canvas for your browser. It is a quiet place to commit the intrusive fragments, loops, and worries living rent-free in your head. It does not exist to solve, organize, or label them — it exists simply to receive them.

It is not a journal. It is not an organizer or a productivity tool. It has no servers, no databases, and no memory. It has no interest in harvesting your data or providing clinical solutions. It is only an ear, vast and silent.

---

## what is stored
Nothing of meaning. The Void remembers your settings (theme, sound toggles) via local storage. If you choose to enable the count ("the void remembers"), it will record the total count of thoughts released and the timestamp of your first and most recent transmissions. Content itself is physically purged from memory the instant you release it.

---

## how to use it
1. Open the page.
2. Type whatever is in your head. Every keystroke has a mechanical voice.
3. Press `Enter` (without Shift) or click `[ ENTER ]` at the bottom. 
4. Watch it dissolve. Feel the whoosh of release. Repeat as needed.

---

## signals
You can tune the atmosphere of your canvas:
- **the expanse** — cold, vast, the quiet between stars.
- **the deep** — ancient, weighted, something older than words.
- **crimson nebula** — slow burn, distant light, something ending, beautifully.

---

## breathing
You can choose how the dark breathes beneath your signals, procedurally synthesized:
- **silence** — just the dark.
- **the drone** — tones. deep. something ancient underneath.
- **the texture** — noise shaped into something vast and soft.
- **both** — the drone inside the dark. the full depth.

---

## tech stack
- **Vite 6** + **React 19** + **TypeScript**
- **Tailwind CSS v4** + **CSS Variables**
- **Web Audio API** (synthesized mechanical clicks, charge pulses, sweeps, and procedural breathing ambience)
- **date-fns** (for minimal relative timekeeping)

---

## running locally

```bash
# 1. install dependencies
npm install

# 2. launch development server
npm run dev

# 3. check typescript types
npm run typecheck

# 4. create production build
npm run build
```
